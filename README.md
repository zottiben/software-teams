## What's new (v0.2.0)

Software Teams specialists are now spawned natively as Claude Code subagents — no more identity-injection preambles. The headline changes:

- **Native subagents** under `.claude/agents/` (generated by `software-teams sync-agents` from `framework/agents/software-teams-*.md`).
- **Doctrine files** `.claude/AGENTS.md` (catalogue) and `.claude/RULES.md` (orchestration / quality), imported by `CLAUDE.md`.
- **Three-tier planning** for non-trivial plans (`SPEC.md` + `ORCHESTRATION.md` + per-agent task slices). `--single-tier` opts back to legacy.
- **Feature flag** `features.native_subagents` (default `true`) in `.software-teams/config/software-teams-config.yaml` for one-flip rollback.

Upgrade an existing install in one command:

```bash
cd <your-software-teams-project>
software-teams sync-agents
```

Full migration guide: [`docs/migration-native-subagents.md`](docs/migration-native-subagents.md).

## Usage Modes

Software Teams runs in three modes with identical capabilities:

| Mode | How | Best For |
|------|-----|----------|
| **Claude Code App** | `/st:create-plan`, `/st:implement-plan`, etc. | Interactive development with Claude Code |
| **CLI** | `software-teams plan`, `software-teams implement`, `software-teams quick`, etc. | Terminal workflows, scripting, CI |
| **GitHub Actions** | Comment `Hey software-teams plan ...` on issues/PRs | Autonomous code generation from issue comments |

## Getting Started

### Claude Code App

```bash
bun install -g @benzotti/software-teams
software-teams init
```

Then inside Claude Code:
```
/st:create-plan "Add user authentication"
/st:implement-plan
/st:commit
/st:generate-pr
```

### CLI

```bash
bun install -g @benzotti/software-teams
software-teams init

software-teams plan "Add user authentication"
software-teams implement
software-teams quick "fix the broken quote calculator"
software-teams review 42
software-teams pr
```

### GitHub Actions

```bash
software-teams setup-action
```

Then on any issue or PR, comment:
```
Hey software-teams plan Add user authentication
Hey software-teams implement
Hey software-teams quick fix the broken login button
```

See [GitHub Actions Setup](#github-actions-setup) for details.

## Commands

| Slash Command | CLI Equivalent | Description |
|---------------|----------------|-------------|
| `/st:init` | `software-teams init` | Initialise Software Teams in the current project |
| `/st:create-plan` | `software-teams plan <desc>` | Create an implementation plan |
| `/st:implement-plan` | `software-teams implement [plan]` | Execute a plan (single-agent or Agent Teams) |
| `/st:quick` | `software-teams quick <desc>` | Quick focused change without orchestration |
| `/st:commit` | `software-teams commit` | Create a conventional commit |
| `/st:generate-pr` | `software-teams pr` | Generate and create a pull request |
| `/st:pr-review` | `software-teams review <pr>` | Review a PR and post line comments |
| `/st:pr-feedback` | `software-teams feedback` | Address PR review comments |
| `/st:worktree` | `software-teams worktree` | Create an isolated git worktree |
| `/st:worktree-remove` | `software-teams worktree-remove` | Remove a worktree and clean up |
| `/st:status` | `software-teams status` | Show current state and suggest next action |

### CLI Flags

#### `software-teams plan`
| Flag | Description |
|------|-------------|
| `--print` | Print prompt to stdout instead of executing |
| `--output <file>` | Write prompt to file |

#### `software-teams implement`
| Flag | Description |
|------|-------------|
| `--team` | Force Agent Teams mode |
| `--single` | Force single-agent mode |
| `--dry-run` | Preview changes without writing files |
| `--print` | Print prompt to stdout |

#### `software-teams quick`
| Flag | Description |
|------|-------------|
| `--dry-run` | Preview changes without writing files |
| `--print` | Print prompt to stdout |

#### `software-teams pr`
| Flag | Description |
|------|-------------|
| `--draft` | Create as draft PR |
| `--base <branch>` | Base branch (default: main) |
| `--no-push` | Skip pushing the branch |
| `--dry-run` | Show generated PR without creating |

## GitHub Actions Setup

### Installation

Run `software-teams setup-action` to generate the workflow file, or manually copy `action/workflow-template.yml` to `.github/workflows/software-teams.yml`.

### Required

| Secret | Required | Description |
|--------|----------|-------------|
| `ANTHROPIC_API_KEY` | Yes | Claude API key |
| `CLICKUP_API_TOKEN` | No | ClickUp integration for ticket-driven development |

### Optional

| Variable | Description |
|----------|-------------|
| `SOFTWARE_TEAMS_AUTH_ENABLED` | Set to `true` to restrict Software Teams to write collaborators |
| `SOFTWARE_TEAMS_ALLOWED_USERS` | Comma-separated list of allowed GitHub usernames |

### Comment Syntax

| Command | Description |
|---------|-------------|
| `Hey software-teams plan <description>` | Create an implementation plan |
| `Hey software-teams implement` | Execute the current plan |
| `Hey software-teams implement --dry-run` | Preview implementation without writing |
| `Hey software-teams quick <description>` | Make a quick change |
| `Hey software-teams review` | Review the current PR |
| `Hey software-teams do <clickup-url>` | Full flow: plan + implement from ticket |
| `Hey software-teams ping` | Check framework status |

### Conversation Flow

1. Comment `Create a plan for ...` to start planning
2. Software Teams posts the plan and asks for feedback
3. Reply with refinement feedback (e.g. "change task 2 ...")
4. Say "approved" or "lgtm" to lock the plan
5. Comment `implement the plan` to execute
6. Reply with follow-up feedback to iterate on the implementation

## Features

### Planning and Implementation

1. **Research** — The planner agent analyses your codebase and gathers context
2. **Plan** — Produces a structured `PLAN.md` with task breakdown, dependencies, and success criteria
3. **Execute** — The complexity router decides single-agent or Agent Teams mode
4. **Verify** — Quality gates run automatically after execution
5. **Commit** — Each task gets its own atomic conventional commit

### Git Worktrees

Full-environment git worktrees for isolated development:

- **Full worktree** (`--worktree`): databases, dependencies, migrations, seeders, web server
- **Lightweight worktree** (`--worktree-lightweight`): dependencies and migrations only

### State Management

| File | Purpose |
|------|---------|
| `.software-teams/config/state.yaml` | Runtime state — position, progress, review status |
| `.software-teams/config/adapter.yaml` | Project-type adapter config (tech stack, quality gates) |
| `.software-teams/config/software-teams-config.yaml` | Global configuration (workflow, models) |
